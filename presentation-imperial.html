How we use AI

AI for efficiency. Deterministic methods for rigour.

I use AI (Anthropic Claude) for specific, bounded tasks — never as a black box, never unsupervised, never for final decisions. Every AI output is reviewed by an analyst before it reaches a client.

Where we use AI
  • Transaction classification — matching line items to GHG Protocol categories when keyword rules don't resolve
  • Supplier risk verification — cross-checking deterministic findings against AI's knowledge to identify registry gaps vs real issues
  • Report language — translating technical findings into client-appropriate language
  • Recommended actions — generating engagement strategies per supplier, reviewed by analyst
Where we deliberately don't
  • Emission factor selection — deterministic, rule-based cascading precision. No AI guessing which factor to use
  • Uncertainty calculation — mathematical, using published Pedigree Matrix GSD factors. No AI approximation
  • Supplier scoring — deterministic scoring from verified data sources. AI verifies, doesn't score
  • QC decisions — statistical sampling with hard gate threshold. Pass/fail is arithmetic, not AI

The principle: AI handles the tasks that benefit from natural language understanding. Everything quantitative — emission factors, uncertainty, scoring, QC — is deterministic and auditable. An auditor can trace every number without understanding AI.
